# Thêm Mã cho Xe/máy — Implementation Plan

> **For agentic workers:** Workflow = **Claude làm schema/migration (chính xác), agy làm validation/action/UI (có mẫu)**. KHÔNG có test framework → verify bằng `npx tsc --noEmit` + `npm run lint` + `npm run build` + psql + render thật. Steps dùng checkbox `- [ ]`.

**Goal:** Thêm trường Mã (code) cho Xe/máy — nullable, unique — như mã vật tư, để xe "là 1 loại mã vật tư" theo ý client. KHÔNG trộn data model.

**Architecture:** `Equipment.code String? @unique` (nullable: xe cũ không có mã vẫn hợp lệ, Postgres unique bỏ qua NULL). validation + action lưu code; form/bảng hiện Mã. Cách quản lý xe (giờ làm) GIỮ NGUYÊN.

**Tech Stack:** Prisma 6 + PostgreSQL (Supabase), Next 16, zod. Migration SQL thủ công.

**Phân vai:** Claude = Phase 1 (schema + migration + verify psql). agy = Phase 2 (validation + action + UI). Claude verify mọi thứ + deploy.

**Branch:** `feat/project-truc`. Spec: `docs/superpowers/specs/2026-06-06-ma-xe-may-design.md`.

### Task 1.1: schema.prisma
- [ ] **Bước 1:** Trong model Equipment, thêm dòng `code String? @unique` (sau `id`):
```prisma
model Equipment {
  id        String         @id @default(cuid())
  code      String?        @unique
  name      String
  type      String?
  plateNo   String?
  note      String?
  createdAt DateTime       @default(now())
  logs      EquipmentLog[]
}
```
- [ ] **Bước 2:** `npx prisma format && npx prisma validate` — valid.
- [ ] **Bước 3:** Commit: `git commit -am "feat(xe-may): schema Equipment.code nullable unique"`

### Task 1.2: Migration SQL
- [ ] **Bước 1:** Tạo `prisma/migrations/20260606150000_equipment_code/migration.sql`:
```sql
-- Thêm Mã cho xe/máy (nullable + unique). Postgres unique index bỏ qua NULL
-- nên nhiều xe chưa-có-mã vẫn hợp lệ. An toàn data live.
ALTER TABLE "Equipment" ADD COLUMN IF NOT EXISTS "code" TEXT;
CREATE UNIQUE INDEX IF NOT EXISTS "Equipment_code_key" ON "Equipment"("code");
```
- [ ] **Bước 2:** Áp local: `npx prisma migrate deploy`. Expected: applied.
- [ ] **Bước 3:** Verify psql:
```sql
SELECT column_name, is_nullable FROM information_schema.columns WHERE table_name='Equipment' AND column_name='code';
SELECT indexname FROM pg_indexes WHERE tablename='Equipment' AND indexname='Equipment_code_key';
```
Expected: code nullable YES; unique index tồn tại.
- [ ] **Bước 4:** `npx prisma generate`. Commit: `git add prisma/migrations && git commit -m "feat(xe-may): migration Equipment.code nullable unique"`
